◆ priority

enum class ar::util::logger::priority
strong

Represents priority of the message and decides whether it is shown or not based on the current logger settings.

Minimum priority can be changed using set_minimum_priority().

Enumerator
EVERYTHING_DEV 

No filter, forces DEBUG verbosity.

EVERYTHING 

Only filters debug messages.

WARNING 

Only shows warnings and all error.

APP_ERROR 

Only shows errors.

CRITICAL 

Only shows critical errors.

NOTHING 

Logger disabled, forces RICH verbosity (to avoid confusion)

void ar::util::logger::configure_based_on_args ( const std::vector< std::string > &  args)

Configure global logger based on command line arguments.

First argument decides verbosity (for ex: "-SIMPLE") while second priority (for ex: "EVERYTHING_DEV"), argument name is 1:1 mapped to enum entry.
